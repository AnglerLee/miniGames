# 미로 찾기 게임 (02-maze) 기획서

## 1. 개요
미로 찾기 게임의 다양성과 완성도를 높이기 위해 맵 난이도를 세분화하고, 시간 제한 및 동적 난이도 조절 시스템을 도입한다.

## 2. 주요 기능

### 2.1 맵 다양화 (Difficulty & Maps)
- **난이도 구성**:
    - **초급 (Easy)**: 10x10 크기 (예상), 간단한 경로.
    - **중급 (Normal)**: 15x15 크기 (예상).
    - **고급 (Hard)**: 20x20 크기 (예상).
- **맵 데이터**:
    - 각 난이도 별 20개, 총 60개의 맵을 `maps.json` 파일로 관리.
    - 게임 시작 시 해당 난이도 풀에서 랜덤으로 맵을 로드.

### 2.2 제한 시간 (Time Limit)
- **타이머 기능**:
    - 게임 시작과 동시에 카운트다운 시작.
    - UI 상단에 남은 시간 표시 (예: 00:30).
    - 시간 초과 시 '실패' 메시지와 함께 게임 오버 (재시도 버튼 표시).

### 2.3 동적 난이도 조절 (Dynamic Retry System)
- **재도전 보너스**:
    - 제한 시간 초과(Time Over)로 인해 재도전(Retry)하는 경우, 제한 시간을 **1초** 증가시킴.
    - 예: 기본 30초 -> 실패 -> 재도전 시 31초 -> 또 실패 -> 32초.
    - 사용자가 클리어할 수 있도록 점진적으로 돕는 장치.

### 2.4 관리자 설정 (Admin Settings)
- **설정 항목**:
    - 난이도별 기본 제한 시간 설정.
    - (선택사항) 맵 강제 선택 (테스트 용).
- **저장**: 설정값은 `localStorage`에 저장하여 유지.

## 3. UI/UX
- **메인 화면**: 난이도 선택 버튼, 게임 시작.
- **게임 화면**: 타이머 시각적 강조 (종료 임박 시 빨간색 등).
- **결과 화면**: 성공/실패 여부, 재시도 시 "시간이 1초 늘어났어요!" 안내 메시지 노출 권장.
