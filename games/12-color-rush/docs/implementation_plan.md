# 12-color-rush(색깔 스피드) 업데이트 계획

## 목표
기존의 단순한 색깔 맞추기 게임을 UX적으로 세련되게 개선하고, 관리자(Admin) 기능을 통해 난이도 조절이 가능하도록 업데이트합니다. 또한 게임 개발 가이드라인(Rule)을 준수하여 아키텍처와 기능을 보강합니다.

## 사용자 리뷰 필요 사항
- **Admin 기능**: 시간 제한(기본 60초), 문제 수(기본 15개), 통과 기준(기본 12개) 설정을 추가합니다.
- **게임 로직**: 
    - 전체 제한 시간 모드를 도입합니다.
    - 시간 초과로 실패 시, "시간 더 받고 재도전" 기능을 제공합니다 (+5초 등).
    - 최고 기록(Clear Time)을 저장하고 표시합니다.
- **UI/UX**: 
    - 타이머를 화면 중앙 상단에 크게 배치합니다.
    - 반응형 디자인을 강화하여 모바일에서도 스크롤 없이 플레이 가능하게 합니다.
    - 애니메이션 효과를 추가하여 생동감을 줍니다.

## 검증 계획

### 자동화 테스트
- 없음 (브라우저 수동 테스트 진행)

### 수동 검증
1. **Admin 설정 확인**:
    - Admin 페이지 접속 -> 제한 시간 30초, 문제 10문제, 통과 8문제로 설정 후 저장.
    - 새로고침 후 설정 유지 확인.
    - '초기화' 버튼 클릭 시 기본값 복원 확인.

2. **게임 플레이 확인**:
    - 게임 시작 시 설정된 시간(30초)으로 타이머 시작 확인.
    - 정답/오답 시 피드백 애니메이션 및 사운드 확인.
    - 10문제 진행 후 통과/실패 화면 정상 출력 확인.

3. **실패 및 재도전 확인**:
    - 시간 초과로 실패 시 "시간 추가(예: +5초) 후 재도전" 버튼 표시 확인.
    - 버튼 클릭 시 게임이 해당 시간만큼 추가된 상태로 재개(또는 리셋 후 시간 증가)되는지 확인.

4. **최고 기록 확인**:
    - 게임 클리어 시 현재 기록이 로컬 스토리지에 저장되는지 확인.
    - 재접속 시 최고 기록이 표시되는지 확인.

5. **UI/UX 확인**:
    - 모바일 화면(좁은 폭)에서 레이아웃 깨짐 없는지 확인.
    - 타이머, 점수판 시인성 확인.
