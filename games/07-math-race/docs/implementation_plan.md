# 구현 계획서 - 07-Math-Race 리팩토링

사용자 요청에 따라 07-Math-Race의 설정 방식과 UI를 대폭 수정하여 단순화하고 몰입도를 높입니다.

## 사용자 검토 필요 사항

> [!IMPORTANT]
> - **난이도 설정 이동**: 게임 내 난이도 선택 버튼을 삭제하고, **관리자 페이지**에서 난이도(Lv.1 ~ Lv.5)를 고정합니다.
> - **단일 시간 설정**: 난이도별 시간이 아닌, **게임 전체 제한 시간** 하나로 통합 관리합니다.
> - **UI 단순화**: 콤보(Streak), 남은 문제 수 등 부가 정보를 화면에서 제거하고 **타이머와 문제 카드**에만 집중하도록 변경합니다.
> - **카드 확대**: 모바일 화면에서 문제 카드가 주가 되도록 크기를 키웁니다.

## 변경 제안 사항

### 1. 관리자 페이지 (`admin.html`, `admin.js`)
- **난이도 선택**: `<select>` 추가 (Lv.1 ~ Lv.5).
- **시간 설정**: `<input type="number">` 하나로 통일 (`timeLimit`).
- 기존 난이도별 시간 설정 UI 제거.

### 2. 게임 UI (`index.html`, CSS)
- **Top Bar**: 홈 버튼, 재도전 버튼 배치.
- **Timer**: 중앙 상단에 크게 배치 (05-Sliding-Puzzle 스타일).
- **Main Area**: 
    - 난이도 선택 버튼 제거.
    - 잡다한 스탯(문제 수, 콤보 등) 제거.
    - **문제 카드**: 화면 중앙에 꽉 차게(적절한 여백 남기고) 배치, 폰트 사이즈 확대.
- **테마**: 기존 테마 유지.

### 3. 게임 로직 (`game.js`)
- **설정 로드**: `math_race_settings`에서 `difficulty`와 `timeLimit` 로드.
- **게임 흐름**:
    - 게임 시작 시 Admin에서 설정된 난이도로 바로 시작.
    - 타이머는 설정된 시간으로 시작.
    - 실패/성공 로직은 유지하되, 화면 표시 정보 줄임.

## 검증 계획

### 수동 검증
1.  **관리자 설정**: 난이도를 Lv.5로, 시간을 60초로 설정 후 저장.
2.  **게임 진입**:
    - 난이도 선택 버튼이 없는지 확인.
    - 바로 Lv.5 문제(나눗셈 등)가 나오는지 확인.
    - 타이머가 60초부터 시작하는지 확인.
    - 화면에 타이머와 문제 카드만 시원하게 보이는지 확인.
