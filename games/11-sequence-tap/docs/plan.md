# 11-sequence-tap 게임 개선 계획서

## 1. 개요
기존 '11-sequence-tap' 게임을 사용자 규칙(Game Dev Guide)에 맞춰 개선합니다. 관리자 설정을 통한 난이도 조절, UI/UX 개선, 그리고 동적 재시도 로직을 구현합니다.

## 2. 작업 목표
- **문서화**: `docs` 폴더 생성 및 계획서 작성.
- **관리자 모드 (Admin)**:
    - 게임 난이도 설정 (제한 시간, 숫자 개수) 추가.
    - `localStorage`를 이용한 설정 저장 및 로드.
- **게임 로직 (Game)**:
    - 관리자 설정 연동 (동적 제한 시간 및 숫자 개수).
    - `alert()` 제거 및 커스텀 모달 적용 (Premium UX).
    - **동적 재시도 로직**: 시간 초과로 실패 시, 시간을 조금 더 주어(예: +5초) 재도전할 수 있는 기능 구현.
    - 최고 기록(Best Record) 저장 기능 추가.
- **UI/UX**:
    - 타이머 시인성 강화 (화면 상단 중앙).
    - 반응형 디자인 점검 (모바일 단일 화면).

## 3. 상세 구현 계획

### 3.1. 파일 구조 변경
- `games/11-sequence-tap/docs/plan.md`: 본 계획서. (생성 완료)

### 3.2. 관리자 페이지 (`admin.html`, `admin.js`)
- **UI 변경**:
    - 제한 시간 입력 필드 (기본 20초).
    - 숫자 개수 입력 필드 (기본 20개).
- **로직 변경**:
    - 설정 저장 시 `treasureHunt_gameConfigs`의 해당 게임 설정에 `customSettings` 객체 추가 저장.
    - 예: `customSettings: { timeLimit: 30, totalNumbers: 25 }`

### 3.3. 게임 페이지 (`game.js`)
- **초기화 로직**:
    - `localStorage`에서 설정을 불러와 `TIME_LIMIT`, `TOTAL_NUMBERS` 변수에 적용.
- **실패 시 로직 개선**:
    - 기존 `showFailScreen` (새로고침) 대신, 커스텀 실패 모달 구현.
    - "재도전" 버튼 클릭 시 `retryCount`를 증가시키고, `TIME_LIMIT + (retryCount * 5)` 초로 게임 재시작.
- **성공 시 로직 개선**:
    - `alert()` 제거.
    - `showSuccessScreen` 호출 전 커스텀 결과 모달을 보여주거나, 바로 성공 화면으로 전환하되 소요 시간을 명시.
    - 로컬 스토리지에 '최고 기록(최단 시간)' 저장 및 표시.

## 4. 예상 시나리오
1. 관리자가 제한 시간을 15초, 숫자 개수를 20개로 설정 후 저장.
2. 사용자가 게임 진입 시 15초 카운트다운 시작.
3. 사용자가 시간 초과로 실패 -> "시간이 부족했나요? 조금 더 여유롭게 재도전해보세요! (+5초)" 팝업 등장.
4. 재도전 클릭 시 20초로 게임 재시작.
5. 성공 시 "Clear!" 모달과 함께 소요 시간 표시.
