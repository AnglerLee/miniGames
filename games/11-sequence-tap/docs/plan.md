# 11-sequence-tap UI/UX 개선 계획

## 1. 계획 개요
사용자 피드백에 따라 게임 영역을 확장하고, 숫자가 겹쳐 보이는 문제를 해결하기 위해 버튼 크기를 동적으로 조절하는 로직을 구현합니다.

## 2. 변경 사항

### 2.1 CSS 수정 (`index.html`)
- **게임 컨테이너 확장**: `max-width`를 `500px` -> `600px`로 변경.
- **탭 영역 확장**: `max-height`를 `500px` -> `600px`로 변경하여 더 넓은 공간 확보.

### 2.2 게임 로직 수정 (`game.js`)
- **버튼 크기 동적 계산**:
  - 기본 크기: `55px` (기존 유지)
  - 숫자 개수 20개 초과 시: `50px`
  - 숫자 개수 25개 초과 시: `45px`
- **충돌 로직 연동**: 변경된 버튼 크기에 맞춰 `btnRadius` 및 `minDistance`가 자동으로 계산되도록 수정.
- **반경 계산**: 줄어든 버튼 크기에 맞춰 경계 체크 로직도 자연스럽게 적용됨.

## 3. 검증 계획
1. **레이아웃 확인**: 브라우저 창을 늘려서 게임 영역이 이전보다 넓어졌는지(600px 폭) 확인.
2. **동적 사이즈 확인**:
   - 관리자 모드에서 '숫자 개수'를 30개로 설정.
   - 게임 시작 시 원의 크기가 평소(적을 때)보다 작아진 것을 시각적으로 확인.
   - 10개로 설정했을 때는 원래 크기로 나오는지 확인.
3. **겹침 확인**: 30개 생성 시 번호 식별이 어려울 정도로 겹치는 경우가 줄어들었는지 확인.
