# 11-sequence-tap 버그 수정 계획

## 1. 버그 분석

### 1.1 타이머 카운트다운 오류
- **현상**: 타이머가 0에서 멈춰있다가 나중에 숫자가 바뀌거나, 시작 시 초기 시간이 표시되지 않음.
- **원인**: `startGame()` 함수에서 `startTimer()`를 호출하지만, `timerEl`의 텍스트를 초기 시간(`timeLeft`)으로 즉시 업데이트하지 않음. HTML 초기값이 `0`으로 되어 있어 첫 1초간 0으로 표시됨.
- **해결**: `startGame()` 시작 시 `timerEl.textContent`를 즉시 업데이트.

### 1.2 모바일 생성 오류 (원 부족 현상)
- **현상**: 모바일 등 좁은 화면에서 설정된 개수(예: 30개)만큼 원이 생성되지 않음.
- **원인**: `createButtons` 함수 내에서 위치 선정 실패 시(150회 시도 초과) 실행되는 Fallback 로직에서 `padding` 변수를 사용하는데, 해당 변수가 이전 코드 수정 시 삭제되어 `ReferenceError`가 발생함. 이로 인해 루프가 중단됨.
- **해결**: `createButtons` 함수 내에 `padding` 상수를 다시 정의하거나(예: 30px), Fallback 로직을 수정.

## 2. 수정 계획

### 2.1 `game.js` 수정
- **`startGame` 함수**:
  - `timerEl.textContent = timeLeft;` 추가.
- **`createButtons` 함수**:
  - `const padding = 30;` 정의 추가 (함수 상단).

## 3. 검증 계획
- **타이머 테스트**: 게임 시작 직후 타이머가 `20` (또는 설정값)으로 표시되는지 확인.
- **모바일/생성 테스트**:
  - 관리자 설정에서 '숫자 개수'를 30개로 설정.
  - 브라우저 창을 모바일 크기로 줄임.
  - 게임 시작 시 30개의 원이 모두 생성되는지(에러 없이) 확인.
